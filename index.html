Persistent and Asynchronous Multicast System

  • ****

Distributed Computing Systems | Spring 2021
  • A system for supporting persistent asynchronous multicast on top of TCP. This is based on a coordinator-participant paradigm. This project implements both the mutlicast Coordinator and Participants.
  • Implements temporally-bound persistence, in which if a participant is offline more than some specified time threshold (say t), upon reconnecting, it will receive messages that were sent in the past t seconds.
  • Multicast participant supports a UI thread to register and deregister, along with sending multicasting messages.
  • Technology Stack: Java, Linux

DOH-Capable DNS Forwarder

  • ****

Computer Networks | Fall 2020
  • This project uses Python to build a simple DNS forwarder with domain blocking and DoH capabilities. This DNS forwarder receives an arbitrary DNS message from a client, checks if the domain name should be blocked, and if so responds with an NXDomain message. If the queried domain name is allowed, it forwards the DNS message to either a standard DNS resolver or a DoH-capable resolver and wait for the response from the resolver and forward it back to the client.
  • Scapy module used for network communication programming.
  • Technology Stack: Pytohn, Linux, Scapy
